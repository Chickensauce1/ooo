# 参与贡献

感谢你愿意改进词尾接接乐。提交 Issue 或 Pull Request 前，请先搜索是否已有相同问题。

## 开发流程

1. Fork 仓库并从默认分支创建短期功能分支。
2. 使用 `npm install` 安装依赖。
3. 保持改动聚焦，不要在同一个 Pull Request 中混入无关重构。
4. 运行 `npm test`，确保规则和词库验证全部通过。
5. 如果修改界面，请同时检查桌面和移动宽度下的显示效果。
6. 在 Pull Request 中说明改动目的、验证方法和用户可见影响。

## 词库修改

- 保持单词为小写英文字母，不加入重复条目。
- 中文释义应简洁、准确，并与目标学习阶段相符。
- 修改生成的官方词库前，优先更新其数据源或构建逻辑。
- 提交前运行 `npm test`，验证接龙规则、词库唯一性和构建器输出。

## 问题反馈

报告问题时，请提供操作系统、应用版本、复现步骤、预期行为和实际行为。不要在公开 Issue 中提交密码、令牌或其他敏感信息。
